Product overview

FlowTerra — what we're building

An AI-powered B2B lead generation and outreach platform. Three autonomous agents work in sequence to scrape, qualify, message, and book meetings — with no manual involvement required from the client's sales team.

Core value proposition
Most lead gen tools create volume — not quality. FlowTerra's differentiator is that the Premium tier's Agent 3 gatekeeps the calendar: no weak lead books a call. Sales reps only show up to conversations worth having.

Problem statement

The problem we're solving

⚠
Core business risk: Without strong lead qualification, the platform floods sales teams with calls that cannot convert. This wastes rep time, creates churn, and undermines trust in the product. The Premium tier's entire value is in preventing this.
Problem
Traditional outreach tools optimise for volume. High quantity of leads ≠ high quality of booked calls.
Impact
Sales reps waste time on calls with leads who cannot buy, are outside the target market, or have low intent.
Solution
Agent 3 acts as a gatekeeper. It manages the conversation, assesses intent, and only books calls that meet the client's ICP threshold. Standard tier users qualify manually.
Messaging
"Only pay for leads worth showing up to" — this line should be prominent in hero, pricing, and any sales collateral.

Audience

Who we're building for

Primary buyer
B2B business owners or sales leads who run outbound sales and want to remove manual prospecting from the equation.
Examples
HVAC companies, digital agencies, SaaS sales teams, facilities management, local service businesses scaling outbound
Pain point
Too much time on prospecting and cold outreach. Not enough time on closing.
Purchase trigger
Tired of wasted sales calls. Want predictable pipeline without hiring more SDRs.
ℹ
The platform is niche-agnostic — it should work for any B2B vertical. The niche configurator on the dashboard lets clients set their target industry, location, and ICP weights. Build all UI with configurability in mind, not hardcoded to any single vertical.

Agent architecture

Three agents, one pipeline

Each agent has a defined input, process, and output. They run sequentially. Agent 3 (Premium only) is the quality gate that solves the weak-leads problem.

Agent 1 — Discovery
Finds and scores leads matching the client's ICP. Runs on-demand or on a schedule.
Input
Niche config (business type, location, radius). ICP weights set by client.
Primary source
Google Places API — returns business name, address, phone, category, rating, review count.
Fallback source
Web scraper (Make.com workflow) — activates when Places API returns insufficient results.
Enrichment
Apollo.io — adds employee count, LinkedIn URL, email, revenue estimate, tech stack signals.
ICP scoring
6 criteria, client-editable weights (must sum to 100%). Score: 0–100. Threshold configurable. Leads below threshold are filtered out.
Output
Scored lead list stored in GHL CRM. Passed to Agent 2 automatically.
// ICP scoring criteria (example — all editable in UI) criteria: [ { label: "Business size", weight: 20 }, { label: "Location fit", weight: 20 }, { label: "Review volume", weight: 15 }, { label: "Online presence", weight: 15 }, { label: "Industry relevance", weight: 20 }, { label: "Contact availability", weight: 10 } ] // weights must sum to 100
Agent 2 — Outreach UI: Next to build
Runs multi-step SMS outreach sequences on leads passed from Agent 1. Handles timing, personalisation, and follow-ups automatically.
Input
Scored lead list from Agent 1 (above ICP threshold only).
SMS provider
Twilio (client's own account) or LeadConnector (via GHL). Per-message cost: $0.0125/segment.
AI layer
OpenRouter → Claude Sonnet (anthropic/claude-sonnet-4). Generates personalised message copy per lead. Cost: $0.018/AI response.
Sequence
Multi-step: initial message → follow-up 1 (Day 2) → follow-up 2 (Day 5) → final (Day 9). Each step personalised. Sequence pauses on reply.
Output
Replied/engaged leads flagged and passed to Agent 3. Non-responders stay in nurture.
ℹ
Agent 2 UI needs: sequence builder (step editor, delay settings, message template with AI generation), live outreach stats panel, per-lead status tracker (sent / delivered / replied / opted out).
Agent 3 — Booking & qualification Premium only
The quality gate. Manages live AI conversations with engaged leads. Qualifies intent before booking. Blocks weak leads from reaching the calendar.
⚠
This is the core differentiator. Without Agent 3, clients receive raw responses and must qualify manually — leading to wasted calls. Agent 3's blocking logic must be prominently explained in UI and marketing copy.
Input
Engaged leads (replied to Agent 2) passed automatically.
AI engine
OpenRouter → Claude Sonnet. Handles objections, asks qualifying questions, assesses intent and fit.
Qualification
Agent evaluates: budget signal, decision-maker status, timeline, service fit. Leads that don't pass are tagged "nurture" — not booked.
Booking
Qualified leads: Agent books directly via Google Calendar API or Calendly embed. Sends confirmation SMS.
CRM sync
All conversation data, qualification score, and booking status synced to GHL. GHL triggers follow-up automations.
Output
Booked calendar appointments for qualified leads only. Nurture sequence for warm-but-not-ready leads.

All business-type targeting, ICP weights, and location settings must be configurable from the dashboard UI. No hardcoded verticals.
